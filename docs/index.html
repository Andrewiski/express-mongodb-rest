api
Parameters
options (Object = {}) options for this function.
Name Description
options.mongodb Object (default {}) default options for MongoDB database.
options.mongodb.connection string (default process.env.MONGODB_CONNECTION||'mongodb://localhost:27017') MongoDB connection string .
options.mongodb.database string (default process.env.MONGODB_DATABASE||'test') database name.
options.mongodb.collection string (default process.env.MONGODB_COLLECTION||'express_mongodb_rest') collection name
options.mongodb.method string (default process.env.MONGODB_METHOD||'find') MongoDB collection method name
options.mongodb.query (Object | string) (default process.env.MONGODB_QUERY||{}) base query when URL query string is not provided (such as localhost:3000/api ) for collection method defined by options.mongodb.method
options.mongodb.callback (function | string) (default process.env.MONGODB_CALLBACK||function(args,result){return(results);}) callback function before sending the response and after querying the MongoDB database
  • Callback is in the form of function(args, result) {}
  • Callback must return a resulting object from a MongoDB collection call
  • args is an Array of arguments passed to the MongoDB collection method defined by options.mongodb.method
  • result is the returned object from the MongoDB collection method defined by options.mongodb.method
  • args[0] is the parsed JSON query from the URL request, where 0 refers to options.mongodb.position
  • This callback is useful to add forced calls such as: function(args, result){return result.limit(1000);}
options.mongodb.args (Array | string) (default process.env.MONGODB_ARGS||[]) Array of arguments to pass to the MongoDB collection method defined by options.mongodb.method
options.mongodb.position (number | string) (default process.env.MONGODB_POSITION||0) position of parsed query JSON from URL request in options.args
options.rest Object (default {}) options for REST API definitions
  • Each key in options.rest is the REST API method such as GET, POST, PUT, DELETE, etc
  • GET is used as an example below, but can be changed to POST, PUT, DELETE, etc
options.rest.GET Object (default {}) example of REST API definition for GET
options.rest.GET.database string (default options.mongodb.database) database name for GET request
options.rest.GET.collection string (default options.mongodb.collection) collection name for GET request
options.rest.GET.method string (default options.mongodb.method) method name for GET request
options.rest.GET.query Object (default options.mongodb.query) base query when URL query is not provided for collection method defined by options.rest.GET.method
options.rest.GET.callback function (default options.mongodb.callback) callback function for GET request as defined in options.mongodb.callback
options.rest.GET.args Array (default options.mongodb.args) Array of arguments to pass to MongoDB collection method defined by options.rest.GET.method
options.rest.GET.position number (default options.mongodb.position) position of parsed JSON from URL request for options.rest.GET.args
Returns
function: Express middleware compatible with app.use .
Example
var express = require('express');
var api = require('express-mongodb-rest');

// (options) Initialize options object
var options = {mongodb: {}, rest: {}};

// (connection_mongodb) Setup mongodb connection
// Format: 'mongodb://<user>:<password>@<host>:<port>'
options.mongodb.connection = 'mongodb://localhost:27017';

// (options_get) Setup REST options for GET
options.rest.GET = {};
options.rest.GET.database = 'test';
options.rest.GET.collection = 'express_mongodb_rest';
options.rest.GET.method = 'find';

// (options_get_limit) Limit documents returned by GET to 100
// Note: args[0] is the parsed query JSON object from the url request
options.rest.GET.callback = function(args, result){return result.limit(100);};

// (app) Create express app with middleware
// Available at: localhost:3000/api
var app = express();
app.use('/api', api(options);
app.listen(3000);
